# 安全政策

## 支持版本

Socrate 目前是早期原型，暂不维护多版本安全分支。请默认使用 `main` 分支最新代码。

## 报告漏洞

请不要在公开 Issue 中披露可利用漏洞、密钥、私有服务器信息或用户数据。

推荐报告内容：

- 漏洞影响范围
- 最小复现步骤
- 可能的攻击路径
- 已验证的平台和版本
- 建议修复方式

## 密钥处理

`backend/.env` 只用于本地运行，已被 `.gitignore` 排除。任何真实 API Key、私有域名、服务器 IP、SSH 端口都不应提交到公开仓库。

如果发现密钥被提交：

1. 立即吊销泄露密钥。
2. 生成新密钥。
3. 清理 git 历史中的敏感内容。
4. 检查访问日志。
